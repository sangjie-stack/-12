# 第一阶段结果与分析草稿

## 结果与分析

本阶段围绕乐高积木识别项目的数据准备与工程初始化展开，已经完成了项目标准目录结构搭建、依赖文件配置、原始数据整理、数据质量检查、图片预处理和数据集划分等工作。项目目前已经具备较完整的第一阶段实验基础，能够为后续模型训练和网页应用开发提供稳定的数据与代码支撑。

在数据准备方面，当前原始数据集根目录为 `data/raw`，共包含 `7` 个基础 Brick 类别，分别为 `1x1`、`1x2`、`1x3`、`1x4`、`2x2`、`2x3`、`2x4`。在重新加入新拍摄图片并统一命名后，数据集有效图片总数达到 `168` 张，其中网站采集图片 `120` 张，新增实拍图片 `48` 张。经过数据质量检查后，结果表明当前数据集无缺失类别、无额外类别、无无效文件，说明本阶段的数据整理工作较为规范，能够满足后续实验的基本要求。

从各类别分布情况来看，`1x1` 类共有 `25` 张，`1x2` 类共有 `27` 张，`1x3` 类共有 `3` 张，`1x4` 类共有 `32` 张，`2x2` 类共有 `23` 张，`2x3` 类共有 `24` 张，`2x4` 类共有 `34` 张。可以看出，当前数据量较之前进一步增加，类别也从原来的 6 类扩展到了 7 类，其中新增的 `1x3` 为后续扩展模型类别提供了基础。虽然 `1x3` 当前样本数量还较少，但项目已经在数据层面支持了这一新规格。

在数据质量方面，本次统计结果显示图像格式包含 `PNG` 和 `JPEG` 两类，其中 `PNG` 图片 `120` 张，`JPEG` 图片 `48` 张；图像颜色模式主要为 `RGB`，也包含少量 `RGBA` 图片。原始图像宽度范围为 `146` 到 `2400` 像素，高度范围为 `141` 到 `2400` 像素，说明原始数据在分辨率和来源上具有一定差异。因此，本项目进一步使用 Pillow 对原始图片进行了统一的 `64x64` 缩放和标准化命名处理，并将结果输出到 `data/processed` 目录中，从而保证后续训练阶段输入尺寸一致，减少因图片大小不一致带来的干扰。

在数据集划分方面，项目已按照 `70% / 15% / 15%` 的比例重新完成训练集、验证集和测试集划分，划分结果保存于 `data/splits` 目录中。当前训练集共有 `116` 张，验证集共有 `27` 张，测试集共有 `25` 张。具体来看，训练集各类别数量分别为 `17`、`19`、`1`、`22`、`16`、`17`、`24`；验证集各类别数量分别为 `4`、`4`、`1`、`5`、`4`、`4`、`5`；测试集各类别数量分别为 `4`、`4`、`1`、`5`、`3`、`3`、`5`。该划分结果整体上保持了类别分布的基本均衡，但也说明新增的 `1x3` 类样本仍偏少，后续若要将其正式纳入模型训练，建议继续补充实拍图片。

从工程化角度来看，本阶段还完成了项目标准目录结构搭建，包括 `model/`、`utils/`、`pages/`、`data/`、`assets/` 等主要目录，同时补齐了 `requirements.txt`、`README.md`、数据处理脚本和 Git 版本管理基础。通过这些工作，项目不再只是零散脚本集合，而是形成了较规范的课程实验工程，为第二阶段的 `Dataset` 构建、CNN 模型训练和对比实验提供了清晰的基础框架。

综合来看，第一阶段已经较好完成了“数据准备”和“项目初始化”的目标。尤其是在重新加入实拍图片并扩展 `1x3` 类别之后，数据总量、数据来源和类别覆盖面都得到了进一步增强，使得数据集更贴近真实应用场景。后续仍可继续补充更多不同拍摄角度、不同光照条件下的实拍数据，特别是补充 `1x3` 类样本，以进一步提高数据集的丰富度和模型泛化能力，但从当前实验任务要求来看，第一阶段已经基本完善，并具备继续开展后续模型训练与功能扩展的条件。
