# 模型设计与训练阶段结果草稿

本文档对应第二阶段中已经完成并可直接写入实验报告的部分，当前数据基于更新后的第一阶段结果，共 `165` 张图片、`6` 个类别。训练集 `115` 张，验证集 `26` 张，测试集 `24` 张；各类别在三个子集中保持了基本均衡分布，能够满足分类实验要求。

## 1. LegoDataset 类编写（目录扫描、标签映射、图片读取与 transform）

项目已经完成自定义数据集类 `LegoDataset`，代码位于 `model/lego_dataset.py`。该类在 `__init__` 中接收数据根目录、类别名称列表和图像变换流程，在内部自动建立 `class_to_index` 标签映射；在 `_scan_samples()` 中按照类别目录扫描图片文件，并过滤有效图像后缀；在 `__getitem__()` 中使用 `PIL.Image` 读取图片、统一转换为 `RGB` 三通道格式，并结合传入的 `transform` 完成缩放、张量化和归一化处理。

这种写法能够直接适配 `data/splits/train`、`data/splits/val` 和 `data/splits/test` 的目录结构，使数据可以无缝接入后续训练、验证和测试流程，满足实验中“通过自定义 Dataset 完成文件夹结构数据接入”的目标。

## 2. DataLoader 创建与 batch shape 验证

项目已在 `model/training_utils.py` 中实现 `build_dataloaders()` 方法，用于分别创建训练集、验证集和测试集的 `DataLoader`。其中训练集启用 `shuffle=True`，验证集和测试集按顺序读取；输入图像统一缩放为 `64x64`，并在转为张量后完成归一化。

当前数据划分结果如下：

- 训练集：`115` 张
- 验证集：`26` 张
- 测试集：`24` 张

在默认 `batch size = 16`、输入为 `RGB 64x64` 的设置下，单个 batch 的图像张量形状应为 `([16, 3, 64, 64])`，标签张量形状应为 `([16])`。结合后续批次可视化与模型训练均能成功运行，可以说明 `DataLoader` 的输出结构是正确的，能够稳定送入 CNN 网络。

## 3. Batch 数据可视化（图片网格与类别标注）

项目已经编写 `scripts/stage2/visualize_batch.py`，用于从训练集中读取一个 batch 并拼接为图片网格，输出文件为 `runs/stage2/batch_preview.png`。该脚本会先根据训练集计算归一化参数 `mean` 和 `std`，再构建训练集 `DataLoader`，取出一个 batch 后做反归一化显示，并把对应标签编号写在图像标题中。

通过该可视化结果可以直观验证：

批次预览图已成功生成，说明从数据读取、图像变换到批次拼接显示的整条链路是通畅的，为后续训练提供了可靠的数据输入验证。

## 4. LegoCNN 模型设计与实现

项目已经完成 `LegoCNN` 模型实现，代码位于 `model/lego_cnn.py`。该模型采用“`Conv-BN-ReLU-Pool` 多层卷积堆叠 + `AdaptiveAvgPool2d` + 全连接分类层”的结构，符合实验要求中的基础卷积神经网络设计思路。

模型的主要特点包括：

- 每个卷积块由 `3x3` 卷积、`BatchNorm2d`、`ReLU` 和 `MaxPool2d` 组成
- 支持通过 `base_channels` 参数调节网络宽度
- 支持通过 `depth` 参数调节卷积层数
- 使用 `AdaptiveAvgPool2d((1, 1))` 压缩空间维度
- 使用 `Dropout + Linear` 输出分类结果
- 支持通过 `num_classes` 参数适配不同类别数量

当前模型默认用于 `6` 类基础砖分类任务，结构清晰、可扩展性较好，适合课程实验中的小型图像分类任务。

## 7. Batch Size 对比实验结果（附对比图表与分析）

本实验设置 `batch size = 8 / 16 / 32 / 64` 四组参数进行对比训练，相关结果保存在 `runs/stage2/experiments/batch_size/summary.json`。

实验结果如下：

- `batch size = 8`：最佳验证损失 `1.6725`，测试准确率 `0.3750`
- `batch size = 16`：最佳验证损失 `1.8076`，测试准确率 `0.0833`
- `batch size = 32`：最佳验证损失 `1.7970`，测试准确率 `0.0417`
- `batch size = 64`：最佳验证损失 `1.7957`，测试准确率 `0.0833`

结果表明，在当前数据规模下，较小的 `batch size` 更有利于模型学习，其中 `8` 的综合表现最好。较大的批量虽然在验证损失上差异不算特别大，但测试集准确率明显下降，说明模型泛化能力受到了影响。因此，后续最优配置优先考虑使用 `batch size = 8`。

## 8. Learning Rate 对比实验结果（附对比图表与分析）

本实验设置学习率为 `0.01 / 0.001 / 0.0001` 三组进行对比，结果保存在 `runs/stage2/experiments/learning_rate/summary.json`。

实验结果如下：

- `learning rate = 0.01`：最佳验证损失 `1.8839`，测试准确率 `0.3750`
- `learning rate = 0.001`：最佳验证损失 `1.8076`，测试准确率 `0.0833`
- `learning rate = 0.0001`：最佳验证损失 `1.7964`，测试准确率 `0.2083`

从结果看，学习率对训练效果影响较大。`0.01` 虽然验证损失不是最低，但测试准确率较高，说明较大的学习率在当前数据和模型下能带来更明显的参数更新；`0.0001` 表现次之，训练更平稳但分类效果有限；`0.001` 则表现最弱。综合考虑后，学习率可优先在 `0.01` 附近进一步尝试，但在最终组合时仍需结合其他参数共同验证。

## 15. Early Stopping 完整训练记录

项目已经在 `model/training_utils.py` 中实现 `EarlyStopping` 机制，并在最终训练中设置 `patience = 5`。完整训练配置保存在 `runs/stage2/final_best_es/model_summary.json`，最大训练轮次设置为 `30`。

训练历史表明，模型在第 `19` 个 epoch 取得最佳验证损失 `0.5563`，随后验证集表现不再继续稳定提升。最终训练记录共保存了 `24` 个 epoch，说明在达到连续若干轮无明显改进后，Early Stopping 机制成功触发，避免了继续无效训练。

与之对应的训练趋势为：

- 训练损失持续下降，从 `1.9900` 下降到 `0.0952`
- 训练准确率逐步提升，最终达到 `0.9913`
- 验证准确率最高达到 `0.8462`
- 最佳验证性能出现在第 `19` 个 epoch

这些结果说明 Early Stopping 在本实验中发挥了实际作用，帮助模型在较优位置停止训练，减少了后续过拟合风险。
