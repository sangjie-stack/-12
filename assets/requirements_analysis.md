# 乐高积木识别项目需求分析

## 项目背景

项目目标是对常见乐高基础砖块进行图像识别，并为后续模型训练和网页应用开发准备统一的数据和代码结构。

## 功能需求

1. 支持整理 6 类基础砖块图片：`1x1`、`1x2`、`1x4`、`2x2`、`2x3`、`2x4`。
2. 支持检测无效图片、异常格式和尺寸信息统计。
3. 支持将原始图片统一缩放到 `64x64` 并按照统一命名规则输出。
4. 支持按 `70% / 15% / 15%` 比例生成训练集、验证集和测试集。
5. 支持 Git 管理项目代码，便于后续功能迭代。

## 非功能需求

1. 脚本运行方式尽量简单，便于课堂实验复现。
2. 输出目录结构清晰，便于后续训练代码直接读取。
3. 支持在 Windows 环境下直接运行。

## 用户场景

1. 学生将不同类别的乐高图片放入 `data/raw/类别名/` 目录。
2. 先运行质量检查脚本，确认数据可用。
3. 再运行缩放和重命名脚本，得到规范化数据。
4. 最后运行分层划分脚本，得到训练集、验证集和测试集。
